# ПЗ-11: Управление доступом на основе ролей (RBAC)

---

## Описание

Реализована система управления доступом на основе ролей (Role-Based Access Control - RBAC) для веб-приложения E-Shop. Система поддерживает 4 уровня доступа:

- **Гость** — не аутентифицированный пользователь
- **Пользователь** — просмотр товаров и поиск по id
- **Продавец** — добавление и редактирование товаров
- **Администратор** — управление товарами и пользователями

## Что реализовано

✅ **Аутентификация и авторизация:**
- JWT access tokens (15 минут) и refresh tokens (7 дней)
- Middleware проверки ролей (`rolesMiddleware`)
- Четыре уровня доступа (Guest, User, Seller, Admin)

✅ **Бэкенд:**
- API маршруты с ограничением доступа по ролям
- `/api/users` маршруты для администраторов
- Управление ролями пользователей
- Удаление (блокировка) пользователей

✅ **Фронтенд:**
- React-фронтенд с экранами входа и регистрации
- Выбор роли при регистрации
- Таблица управления пользователями (для администраторов)
- Условное отображение элементов UI на основе роли
- Axios interceptors для автоматического обновления токенов
- CRUD товаров
- Поиск товара по id

## API маршруты

### Аутентификация (Гость)

| Метод | Маршрут              | Описание                          |
|-------|----------------------|-----------------------------------|
| POST  | `/api/auth/register` | Регистрация с выбором роли        |
| POST  | `/api/auth/login`    | Вход — возвращает access/refresh  |
| POST  | `/api/auth/refresh`  | Обновление токенов               |
| GET   | `/api/auth/me`       | Текущий пользователь (требует JWT)|

### Товары

| Метод  | Маршрут              | Роль             | Описание               |
|--------|----------------------|------------------|------------------------|
| GET    | `/api/products`      | Все              | Список товаров        |
| GET    | `/api/products/:id`  | User+            | Получить товар по id  |
| POST   | `/api/products`      | Seller+          | Создать товар         |
| PUT    | `/api/products/:id`  | Seller+          | Обновить товар        |
| DELETE | `/api/products/:id`  | Admin            | Удалить товар         |

### Пользователи (Администратор)

| Метод  | Маршрут       | Описание                    |
|--------|---------------|-----------------------------|
| GET    | `/api/users`  | Список пользователей        |
| GET    | `/api/users/:id`  | Получить пользователя      |
| PUT    | `/api/users/:id`  | Обновить роль пользователя |
| DELETE | `/api/users/:id`  | Удалить пользователя       |

## Roles Middleware

```javascript
function rolesMiddleware(...allowedRoles) {
  return (req, res, next) => {
    if (!req.user || !allowedRoles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Forbidden: insufficient permissions' });
    }
    next();
  };
}
```

**Примеры использования:**
```javascript
// Только для продавцов и администраторов
app.post('/api/products', authMiddleware, rolesMiddleware('seller', 'admin'), handler);

// Только для администраторов
app.delete('/api/users/:id', authMiddleware, rolesMiddleware('admin'), handler);
```

## Контроль доступа на фронтенде

**Кнопка "Добавить товар":**
- Видна только для `seller` и `admin`

**Кнопка "Пользователи":**
- Видна только для `admin`

**Редактирование товара:**
- Видно только для `seller` и `admin`

**Удаление товара:**
- Видно только для `admin`

**Таблица пользователей:**
- Доступна только для `admin`
- Можно менять роли: User → Seller → Admin
- Можно удалять пользователей

## Регистрация и выбор роли

При регистрации пользователь может выбрать начальную роль:
- **Пользователь** — просмотр товаров
- **Продавец** — добавление/редактирование товаров

Администратор может быть назначен только уже зарегистрированным пользователем через таблицу управления.

---

## JWT Payload

Access и refresh токены содержат:
```json
{
  "sub": "user_id",
  "email": "user@example.com",
  "role": "seller",
  "iat": 1234567890,
  "exp": 1234568890
}
```

---

## Безопасность

- 🔐 Пароли хешируются bcrypt (10 раундов)
- 🔑 JWT токены подписаны с разными секретами
- ⏱ Access token: 15 минут, Refresh token: 7 дней
- 🛡️ Все защищённые маршруты проверяют роль
- 📍 Статус 403 при недостаточных правах
